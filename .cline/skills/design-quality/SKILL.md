# Design Quality Guide

Figma 디자인 결과물의 품질을 프리미엄 수준으로 끌어올리기 위한 감사 체크리스트와 개선 패턴.
DESIGN.md의 토큰/원칙과 함께 사용한다. 이 스킬은 **진단 → 개선 순서**를 다룬다.

## 감사 프로세스

### 1단계: 스캔
현재 프레임/페이지의 구조를 파악한다:
- 사용 중인 폰트, 컬러 팔레트, 레이아웃 패턴
- 컴포넌트 구조, 간격 체계, 쉐도우 사용 현황

### 2단계: 진단
아래 체크리스트로 문제를 식별하고 기록한다.

### 3단계: 수정
우선순위에 따라 타겟팅된 개선을 적용한다. 전체 재작성보다 점진적 개선을 선호한다.

## 디자인 감사 체크리스트

### Typography 감사
- [ ] **폰트 선택**: Inter만 단독 사용하고 있나? → 한국어 콘텐츠는 Pretendard로 교체
- [ ] **헤드라인 존재감**: 한국어 헤드라인이 충분히 크고 임팩트 있는가? (display 스케일 활용)
- [ ] **한국어 줄간격**: 한국어 텍스트에 적절한 line-height가 적용되었나? (120%~130%)
- [ ] **본문 너비**: 본문 텍스트가 너무 넓지 않은가? (최대 ~640px 권장)
- [ ] **Weight 다양성**: Regular과 Bold만 사용하나? → Medium(500), SemiBold(600) 추가
- [ ] **AI 클리셰 카피**: "혁신적인", "획기적인", "차세대" 등 사용 여부 → 구체적 표현으로 교체
- [ ] **존댓말 일관성**: 합니다/하세요 혼용 여부

### Color & Surface 감사
- [ ] **순수 #000000**: 사용 여부 → `#0A0A0A` 또는 틴티드 다크로 교체
- [ ] **과채도 액센트**: 채도 80% 초과 컬러 여부 → 톤다운
- [ ] **복수 액센트**: 2개 이상 액센트가 경쟁하나? → 1개로 통일
- [ ] **AI 퍼플 그래디언트**: 보라/파란 AI 미학 여부 → 중성 기반으로 교체
- [ ] **Generic 쉐도우**: `rgba(0,0,0,0.3)` 같은 무거운 쉐도우 → 틴티드 쉐도우로 교체
- [ ] **플랫 섹션**: 텍스처 없는 밋밋한 배경 → 노이즈, 메쉬 그래디언트, 패턴 고려
- [ ] **컬러 온도 혼용**: warm gray와 cool gray 혼용 여부

### Layout 감사
- [ ] **3열 동일 카드**: 모든 Feature가 같은 크기 3열인가? → 벤토, 지그재그 등으로 교체
- [ ] **반복 레이아웃**: 인접 섹션이 동일 레이아웃인가? → 각 섹션 다른 패턴 사용
- [ ] **중앙 정렬 편향**: 모든 것이 가운데 정렬인가? → 비대칭, 스플릿 레이아웃 도입
- [ ] **여백 부족**: 섹션 간격이 48px 미만인가? → 최소 48px, 프리미엄은 80px
- [ ] **Max-width 미적용**: 콘텐츠가 화면 가장자리까지 닿나? → max-width 1200px 컨테이너
- [ ] **깊이감 부재**: 모든 요소가 같은 평면인가? → 겹침, z-index, 오프셋 활용
- [ ] **CTA 존재감 부족**: CTA 버튼이 충분히 크고 눈에 띄는가? → 최소 padding 16/32

### Component 감사
- [ ] **카드 단순 구조**: 플랫 사각형 카드만 사용 → Double-Bezel 패턴 검토
- [ ] **버튼 상태 부재**: hover/active 상태 변형이 없는가? → 스케일/색상 변형 추가
- [ ] **태그/뱃지 부재**: 섹션 제목 위 Eyebrow Tag가 있으면 시각적 계층이 풍부해짐
- [ ] **일관성 부족**: 같은 종류의 컴포넌트가 다르게 생겼는가? → 통일

## 수정 우선순위 (Impact 순)

최소 변경으로 최대 시각적 개선을 얻는 순서:

| 순위 | 작업 | Impact | 난이도 |
|------|------|--------|--------|
| 1 | **폰트 교체** (Pretendard 도입) | 즉시 프리미엄 느낌 | 낮음 |
| 2 | **컬러 팔레트 정리** (AI 퍼플 제거, 액센트 통일) | 높음 | 낮음 |
| 3 | **한국어 콘텐츠 개선** (자연스러운 카피, 실감 숫자) | 높음 | 중간 |
| 4 | **여백/간격 확대** (섹션 패딩 증가) | 중간 | 낮음 |
| 5 | **레이아웃 다양화** (섹션 패턴 변경) | 높음 | 중간 |
| 6 | **쉐도우/디테일 정리** (틴티드 쉐도우, 글래스) | 중간 | 낮음 |
| 7 | **프리미엄 컴포넌트 적용** (Double-Bezel, Pill CTA) | 높음 | 중간 |

## 프리미엄 디자인 패턴 (Figma 구현)

### Spatial Rhythm (공간 리듬)
- **매크로 여백**: 섹션 간 패딩 최소 `spacing-12`(48px), 프리미엄은 `spacing-20`(80px)
- **Eyebrow → Heading → Body → CTA 흐름**: 시각적 계층을 항상 유지
- **비대칭 여백**: 좌측 큰 여백 + 우측 작은 여백으로 시선 유도

### Surface & Depth (질감과 깊이)
- 카드에 엘리베이션이 계층을 전달할 때만 쉐도우를 사용한다
- 쉐도우 컬러를 배경 색조에 맞춰 틴팅한다
- 글래스 이펙트: `backdrop-blur` + `border white/10` + `inner shadow`

### 한국어 실감 데이터 (플레이스홀더용)
```
이름: 하윤서, 박도현, 이서진, 김하늘, 정민준, 오예린, 최시우, 한지원
회사: 스텔라랩스, 베리파이, 루미너스, 플로우캔버스, 넥스트비전, 브릿지웍스
직책: 프로덕트 디자이너, 스타트업 대표, 마케팅 리드, 프론트엔드 개발자
지표: 47,200+, 4.87/5.0, 2.3초, 98.7%, 12,847개
```

```

- 위치: 프레임/노드 이름
- 현재: 현재 상태

- 근거: DESIGN.md 토큰 또는 이 가이드 규칙 참조
```
